San Bartolo Live - Procedura rapida (2026-04-12, nuova configurazione)

1) Entrare nel progetto
cd C:\Users\sgior\Desktop\sanbartolo-live

2) Aggiornare repository
git pull origin main

3) Avvio aggiornamento web automatico (consigliato)
avvia-update-web.bat

Alternativa da PowerShell:
powershell -NoProfile -ExecutionPolicy Bypass -File .\update-and-push.ps1 -IntervalSeconds 300

4) Cosa fa update-and-push.ps1
- Scarica snapshot da:
  http://44.3.44.133/webcapture.jpg?command=snap&channel=1
- Aggiorna live/latest.jpg e latest.jpg
- Confronta hash e fa commit/push su main solo se immagine cambiata

5) Test locale pagina
py -m http.server 8080
Aprire: http://127.0.0.1:8080/

6) URL pubblica
https://sgiorgini.github.io/sanbartolo-live/

7) Aggiornamento manuale una tantum (se serve)
mkdir live 2>nul
curl -fsSL "http://44.3.44.133/webcapture.jpg?command=snap&channel=1" -o live/latest.jpg
copy /Y live\latest.jpg latest.jpg
git add live/latest.jpg latest.jpg
git commit -m "Refresh live snapshot"
git push origin main

8) Diagnostica veloce
- Se locale non cambia: controllare che update-and-push.ps1 sia in esecuzione.
- Se web non cambia: verificare nuovi commit su main e fare Ctrl+F5.
- Se ancora cache: aprire https://sgiorgini.github.io/sanbartolo-live/?v=1

Nota workflow GitHub:
- .github/workflows/update-snapshot.yml e' stato reso robusto (retry+timeout).
- Se runner GitHub non raggiunge la camera, lo script locale con push mantiene comunque il web aggiornato.
